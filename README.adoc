= GraphQL Field-based Permissions Policy

== Phase

[cols="^2,^2",options="header"]
|===
|onRequest|onResponse

|-
|X

|===
== Description

You can use the `graphql-field-based-permissions` policy to block or prevent specific fields in the original response content from being returned to the consumer.

== Compatibility with APIM

|===
|Plugin version | APIM version

| 0.0.1         | 4.9.x to latest

|===

== Configuration

|===
|Property |Required |Description |Type |Default

.^|blockList
^.^|X
|A list of fields to block or exclude from the returned payload
^.^|list
^.^|-

.^|responsePolicy
^.^|X
|Specify either BLOCK_RESPONSE or REMOVE_BLOCKED_FIELDS
^.^|string
^.^|BLOCK_RESPONSE

|===

[source, json]
.Sample
----
"graphql-field-based-permissions": {
    "blockList": ["field1", "field2"],
    "responsePolicy": "BLOCK_RESPONSE"
}
----

== Errors

=== HTTP status code

|===
|Code |Message

.^| ```500```
| The body content cannot be transformed.

|===
